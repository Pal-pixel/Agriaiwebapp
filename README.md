# AgriAI — Web

A React website for AI-powered crop disease diagnosis. This is the web
rewrite of the original Expo/React Native app (in the parent folder), built
with **Vite + React + TypeScript**.

## Features

- Upload or capture a crop leaf photo (camera input on mobile browsers).
- Disease detection via the Hugging Face Gradio space `bhargavsavaliya/agri-ai-backend`.
- Treatment details (pesticides, dosage, prevention, economic impact) generated by NVIDIA NIM (`meta/llama-3.1-8b-instruct`), forced into valid JSON via `response_format: json_object`.
- "Ask AI" chat for follow-up farming questions (replies capped at ~50 words to save tokens).
- **Multi-language**: English default, switchable to Hindi + regional languages (Marathi, Gujarati, Bengali, Tamil, Telugu, Kannada, Punjabi). The UI chrome is translated for English & Hindi; AI content (details + chat) is generated in the selected language. Preference is saved on the device.
- **Scan history + token-saving cache**: past diagnoses are saved on the device. Re-opening a known disease loads instantly from cache with **zero API tokens**.
- **Read aloud**: a 🔊 Listen button speaks the summary and chat answers via the browser's speech engine (free, no tokens) in the selected language.

## Develop

```bash
cd web
npm install
npm run dev
```

Then open the printed local URL.

## Build

```bash
npm run build      # type-checks and bundles to dist/
npm run preview    # serve the production build locally
```

## Notes

- The NVIDIA NIM API key lives in `src/lib/nim.ts` and is shipped to the
  browser (same as the original app). For a public deployment, move these
  calls behind a small backend so the key is not exposed.
- Token economy: every NIM call passes a hard `maxTokens` cap (detail page
  ~400, chat ~90) and a short system primer, and prompts request terse output.
  Lower these caps further in `Results.tsx` if you want to stretch credits more.
